# SEMO 개인정보처리방침

시행일: 2026년 7월 7일  
개발자 표시명: 이제민  
문의 이메일: sbsemo91@gmail.com  
앱 패키지명: app.semo.moments

SEMO는 사용자의 사진과 영상을 기기 안에서 선택, 편집, 저장, 내보내기 할 수 있도록 돕는 로컬 우선 영상 일기 앱입니다.

본 개인정보처리방침은 SEMO가 어떤 데이터에 접근, 저장, 사용, 전송 또는 공유할 수 있는지 설명합니다.

## 1. 기본 원칙

SEMO는 사용자가 선택하거나 촬영한 사진·영상 원본을 SEMO 자체 서버로 자동 업로드하지 않습니다.

SEMO의 프로젝트 정보, 캡션, D-day 설정, 편집 구간, 볼륨 설정, 선택한 미디어 참조, 백업 정보 등은 기본적으로 사용자의 기기 내부에 저장됩니다.

다만 SEMO는 앱 안정성 개선과 사용성 분석을 위해 Google Firebase Analytics 및 Firebase Crashlytics를 사용합니다. 이 과정에서 일부 앱 사용 정보, 기기 정보, 오류 정보, 앱 인스턴스 식별자 등이 Google Firebase로 전송될 수 있습니다.

## 2. SEMO가 접근하거나 저장할 수 있는 데이터

SEMO는 앱 기능 제공을 위해 다음 데이터에 접근하거나 저장할 수 있습니다.

### 2.1 사진, 영상, 오디오 데이터

SEMO는 사용자가 선택하거나 촬영한 사진과 영상에 접근할 수 있습니다.

사용 목적은 다음과 같습니다.

- 사용자가 선택한 사진 또는 영상을 프로젝트에 추가
- 영상 구간 편집
- 사진을 영상 형태로 변환
- 편집 결과물 내보내기
- 썸네일, 커버 이미지, 보존 클립 생성
- 사용자가 직접 촬영한 사진 또는 영상 저장
- 영상에 포함된 오디오 처리

SEMO는 사진·영상 원본을 SEMO 자체 서버로 자동 전송하지 않습니다.

단, 사용자가 내보낸 영상, 백업 ZIP, 진단 로그 등을 직접 외부 앱 또는 서비스로 공유하는 경우, 해당 데이터는 사용자가 선택한 외부 앱 또는 서비스로 전달될 수 있습니다.

### 2.2 프로젝트 및 편집 데이터

SEMO는 앱 기능 제공을 위해 다음 정보를 기기 내부에 저장할 수 있습니다.

- 프로젝트 이름
- 프로젝트 유형
- 캡션 또는 날짜별 문구
- D-day 설정
- 선택한 미디어 참조 정보
- 편집 구간 정보
- 클립 순서
- 볼륨 설정
- 회전 및 화면 비율 정보
- 내보내기 설정
- 백업 메타데이터
- 앱 설정값
- 최초 가이드 표시 여부

이 데이터는 기본적으로 사용자의 기기 내부 저장소에 저장되며, SEMO 자체 서버로 자동 전송되지 않습니다.

### 2.3 백업, 복원, 내보내기 파일

SEMO는 사용자의 요청에 따라 다음 파일을 생성하거나 처리할 수 있습니다.

- 내보내기 영상 파일
- 백업 ZIP 파일
- 복원 시 사용자가 선택한 백업 ZIP 파일
- 썸네일 또는 보존 클립
- 사진 변환 영상 파일
- 임시 내보내기 파일

내보내기 영상과 백업 ZIP은 사용자의 요청에 따라 기기의 Downloads/SEMO 폴더에 저장될 수 있습니다. 앱 내 촬영 기능으로 저장한 사진 또는 영상은 사용자의 선택에 따라 갤러리 또는 DCIM/SEMO 폴더에 저장될 수 있습니다. 진단 로그를 저장하는 경우에도 Downloads/SEMO 폴더에 저장될 수 있습니다.

이러한 파일은 앱을 삭제하더라도 기기에 남을 수 있으며, 사용자가 직접 삭제해야 할 수 있습니다.

### 2.4 진단 로그

SEMO는 오류 분석과 문제 해결을 위해 진단 로그를 생성할 수 있습니다.

진단 로그에는 다음 정보가 포함될 수 있습니다.

- 앱 버전
- 기기 모델
- 운영체제 버전
- 앱 실행 상태
- 내보내기 처리 상태
- 오류 메시지
- FFmpeg 처리 정보
- 파일 경로 또는 미디어 참조 정보
- 저장공간 관련 정보

진단 로그는 자동으로 SEMO 자체 서버에 전송되지 않습니다. 사용자가 직접 저장하거나 공유하는 경우에만 외부 앱 또는 개발자에게 전달될 수 있습니다.

진단 로그를 공유하기 전에는 파일 경로, 기기 정보, 오류 정보 등이 포함될 수 있음을 확인해 주세요.

## 3. Firebase를 통한 데이터 수집

SEMO는 앱 품질 개선과 오류 분석을 위해 Google Firebase 서비스를 사용합니다.

### 3.1 Firebase Analytics

SEMO는 Firebase Analytics를 사용하여 앱 사용 현황을 분석할 수 있습니다.

수집될 수 있는 정보는 다음과 같습니다.

- 앱 열기
- 프로젝트 생성, 열기, 삭제
- 클립 추가 또는 삭제
- 내보내기 시작, 성공, 실패
- 백업 또는 복원 실행
- 사용 가이드 열기
- 카메라 촬영 시작, 성공, 실패
- 테마 변경
- 잠금 해제 성공
- D-day 설정 여부
- 앱 버전
- 기기 정보
- 운영체제 정보
- 앱 인스턴스 식별자
- 광고 ID 또는 Android 광고 관련 식별자
- 대략적인 지역 정보

SEMO는 Analytics 이벤트에 프로젝트명, 파일명, 캡션, 사용자가 입력한 텍스트, 미디어 경로, 이메일, 전화번호, 이름 등을 의도적으로 포함하지 않습니다.

현재 SEMO 앱 내부에는 Firebase Analytics 수집을 사용자가 직접 비활성화하는 설정이 없습니다.

### 3.2 Firebase Crashlytics

SEMO는 Firebase Crashlytics를 사용하여 앱 오류와 비정상 종료 정보를 수집할 수 있습니다.

수집될 수 있는 정보는 다음과 같습니다.

- 오류 메시지
- 비정상 종료 정보
- stack trace
- 앱 버전
- 기기 모델
- 운영체제 버전
- 앱 실행 상태
- Crashlytics 설치 식별자
- Firebase 설치 식별자

SEMO는 Crashlytics에 사용자 이름, 이메일, 전화번호, 프로젝트명, 캡션, 파일명, 미디어 경로 등을 의도적으로 전송하지 않습니다.

다만 앱 오류의 특성상 오류 메시지나 stack trace에 일부 기술적 정보가 포함될 수 있습니다.

## 4. 앱 권한 사용 목적

SEMO는 Android 기기에서 다음 권한을 사용할 수 있습니다.

| 권한 | 사용 목적 |
|---|---|
| 사진 및 영상 접근 권한 | 사용자가 선택한 사진·영상을 프로젝트에 추가하고 편집하기 위해 사용 |
| 일부 사진 및 영상 접근 권한 | Android 14 이상에서 사용자가 허용한 미디어만 접근하기 위해 사용 |
| 카메라 권한 | 앱 안에서 사진 또는 영상을 촬영하기 위해 사용 |
| 마이크 권한 | 영상 촬영 시 오디오를 함께 기록하기 위해 사용 |
| 인터넷 권한 | Firebase Analytics, Crashlytics, 외부 오류 신고 링크 등 네트워크 기능을 위해 사용 |
| 네트워크 상태 권한 | 네트워크 상태 확인 및 Firebase SDK 동작을 위해 사용 |
| 광고 ID 관련 권한 | Firebase Analytics 및 Google Play Services 측정 기능에서 사용할 수 있음 |
| 저장소 관련 권한 | 일부 Android 버전에서 사진·영상 접근 또는 촬영 파일 처리를 위해 사용 |

SEMO는 전체 파일 접근 권한인 `MANAGE_EXTERNAL_STORAGE`를 사용하지 않습니다.

SEMO는 알림 권한인 `POST_NOTIFICATIONS`를 사용하지 않습니다.

## 6. 제3자 제공 및 외부 서비스

SEMO는 사용자의 사진·영상 원본, 프로젝트 데이터, 캡션, D-day 설정 등을 SEMO 자체 서버로 자동 전송하지 않습니다.

다만 다음 경우에는 데이터가 외부 서비스로 전달될 수 있습니다.

### 6.1 Google Firebase

SEMO는 Firebase Analytics 및 Firebase Crashlytics를 사용합니다.

이 과정에서 앱 사용 정보, 오류 정보, 기기 정보, 앱 인스턴스 식별자, 광고 ID 관련 정보 등이 Google Firebase로 전송될 수 있습니다.

### 6.2 사용자가 직접 공유하는 경우

사용자가 앱의 공유 기능 또는 기기의 파일 공유 기능을 이용하면 다음 데이터가 사용자가 선택한 외부 앱 또는 서비스로 전달될 수 있습니다.

- 내보내기 영상
- 백업 ZIP
- 진단 로그
- 오류 신고에 첨부하거나 입력한 정보

이 경우 데이터 처리는 사용자가 선택한 외부 앱 또는 서비스의 개인정보처리방침을 따릅니다.

### 6.3 외부 오류 신고 링크

SEMO는 오류 신고 또는 문의를 위해 외부 링크를 열 수 있습니다.

사용자가 외부 양식 또는 이메일을 통해 직접 입력한 내용은 해당 외부 서비스 또는 이메일 서비스의 정책에 따라 처리될 수 있습니다.

## 7. 데이터 보관 및 삭제

### 7.1 기기 내부 데이터

프로젝트 정보, 편집 정보, 캡션, D-day 설정, 보존 클립, 썸네일 등은 사용자의 기기 내부에 저장됩니다.

사용자는 앱 내 프로젝트 삭제, 클립 삭제, 저장공간 관리 기능을 통해 관련 데이터를 삭제할 수 있습니다.

앱을 삭제하면 앱 내부 저장소에 저장된 데이터는 일반적으로 함께 삭제됩니다.

### 7.2 앱 외부에 저장된 파일

다음 파일은 앱 외부 저장소에 저장될 수 있으며, 앱을 삭제해도 남을 수 있습니다.

- Downloads/SEMO 폴더에 저장된 내보내기 영상
- Downloads/SEMO 폴더에 저장된 백업 ZIP
- Downloads/SEMO 폴더에 저장된 진단 로그
- 갤러리 또는 DCIM/SEMO 폴더에 저장된 촬영 파일

이러한 파일은 사용자가 파일 앱, 갤러리 앱 또는 저장공간 관리 기능을 통해 직접 삭제해야 합니다.

### 7.3 Firebase 데이터

Firebase Analytics 및 Crashlytics를 통해 수집된 데이터는 Google Firebase의 보관 정책과 설정에 따라 보관 및 삭제될 수 있습니다.

Firebase 관련 데이터 삭제 또는 문의가 필요한 경우 `sbsemo91@gmail.com`으로 연락해 주세요. 다만 SEMO는 계정 기반 앱이 아니므로 특정 사용자의 Firebase 데이터를 식별하거나 삭제하는 데 제한이 있을 수 있습니다.

## 8. 보안

SEMO는 사용자의 사진·영상과 프로젝트 데이터를 기본적으로 기기 내부에서 처리하도록 설계되어 있습니다.

Firebase로 전송되는 데이터는 Google Firebase SDK를 통해 전송됩니다.

다만 사용자가 내보내기 영상, 백업 ZIP, 진단 로그 등을 외부 앱으로 공유하는 경우, 해당 데이터의 보안과 처리는 사용자가 선택한 외부 앱 또는 서비스의 정책을 따릅니다.

## 9. 아동 개인정보

SEMO는 아동을 직접 대상으로 하는 서비스가 아닙니다.

SEMO는 보호자 또는 일반 사용자가 개인적인 사진·영상 기록을 만들기 위한 앱입니다.

사용자는 아동의 사진이나 영상을 앱에 추가할 수 있으나, 해당 사진과 영상은 기본적으로 사용자의 기기 내부에서 처리되며 SEMO 자체 서버로 자동 업로드되지 않습니다.

## 10. 광고 및 결제

현재 SEMO는 앱 내 광고 SDK 또는 인앱결제 SDK를 사용하지 않습니다.

다만 Firebase Analytics 및 Google Play Services 측정 기능으로 인해 광고 ID 또는 Android 광고 관련 식별자가 사용될 수 있습니다. 이는 현재 SEMO 앱 내 광고 표시 목적이 아니라 앱 사용 분석 및 측정 목적입니다.

향후 광고, 결제, 구독, 계정, 클라우드 동기화 기능이 추가되는 경우 개인정보처리방침을 개정하고 필요한 경우 Google Play Data Safety 신고 내용을 변경합니다.
