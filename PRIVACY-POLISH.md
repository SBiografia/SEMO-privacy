# SEMO 개인정보처리방침

시행일: 2026년 7월 13일

[데이터 삭제 요청 안내](./data-deletion.html)

SEMO는 사용자의 사진과 영상을 기기 안에서 선택, 편집, 저장, 내보내기 할 수 있도록 돕는 로컬 우선 영상 일기 앱입니다.

본 개인정보처리방침은 SEMO가 어떤 개인정보를 처리하는지, 어떤 목적으로 이용하는지, 보유·삭제 방법은 무엇인지, 그리고 Google Firebase 및 Google Play 사용으로 인해 어떤 정보가 외부로 전송될 수 있는지를 설명합니다.

SEMO는 사용자가 선택하거나 촬영한 사진·영상 원본을 SEMO 자체 서버로 자동 업로드하지 않습니다. 사진·영상, 프로젝트 정보, 캡션, D-day 설정, 편집 구간, 볼륨 설정 등은 기본적으로 사용자의 기기 내부에서 처리됩니다.

다만 SEMO는 앱 안정성 개선과 사용성 분석을 위해 Google Firebase Analytics 및 Firebase Crashlytics를 사용합니다. 이 과정에서 일부 앱 사용 정보, 기기 정보, 오류 정보, 앱 인스턴스 식별자 등이 Google Firebase로 전송될 수 있습니다.

## 1. 개인정보의 처리 목적

SEMO는 다음 목적을 위해 필요한 최소한의 범위에서 개인정보 또는 개인정보에 해당할 수 있는 정보를 처리합니다. 처리하는 정보는 아래 목적 외의 용도로 이용하지 않습니다.

| 처리 목적 | 설명 |
|---|---|
| 앱 기능 제공 | 사용자가 선택하거나 촬영한 사진·영상을 프로젝트에 추가하고, 편집 구간을 저장하며, 영상 내보내기 기능을 제공하기 위해 처리합니다. |
| 프로젝트 및 편집 정보 저장 | 프로젝트 이름, 캡션, D-day 설정, 클립 순서, 볼륨 설정, 화면 비율, 내보내기 설정 등을 기기 내부에 저장하기 위해 처리합니다. |
| 백업 및 복원 | 사용자가 요청한 경우 백업 ZIP을 생성하거나, 사용자가 선택한 백업 ZIP을 읽어 앱 내부 데이터를 복원하기 위해 처리합니다. |
| 오류 분석 및 안정성 개선 | Firebase Crashlytics를 통해 앱 오류와 비정상 종료 정보를 확인하고 앱 안정성을 개선하기 위해 처리합니다. |
| 사용 현황 분석 및 기능 개선 | Firebase Analytics를 통해 앱 사용 흐름을 분석하고 기능 개선에 참고하기 위해 처리합니다. |
| 문의 및 오류 신고 대응 | 사용자가 이메일 또는 외부 오류 신고 양식을 통해 직접 제공한 내용을 확인하고 답변하기 위해 처리합니다. |
| 인앱결제 및 Premium 이용권 관리 | Google Play를 통한 Premium 상품 구매 처리, 구매 상태 확인, 유료 기능 활성화, 기존 구매 복원, 중복 구매 방지 및 환불·취소 상태 확인을 위해 처리합니다. |

## 2. 처리하는 개인정보의 항목

SEMO가 처리할 수 있는 개인정보 또는 개인정보에 해당할 수 있는 정보는 다음과 같습니다. SEMO는 프로젝트명, 파일명, 캡션, 사용자가 입력한 텍스트, 미디어 경로, 이메일, 전화번호, 이름 등을 Firebase Analytics 이벤트에 포함하지 않도록 설계·운영합니다.

| 구분 | 처리 항목 | 처리 방식 | 외부 전송 여부 |
|---|---|---|---|
| 사진·영상·오디오 데이터 | 사용자가 선택하거나 촬영한 사진·영상, 영상에 포함된 오디오, 썸네일, 커버 이미지, 보존 클립, 사진 변환 영상 | 기기 내부에서 선택, 편집, 변환, 저장, 내보내기 | SEMO 자체 서버로 자동 전송하지 않음. 사용자가 직접 공유하는 경우 선택한 외부 앱 또는 서비스로 전달될 수 있음 |
| 프로젝트 및 편집 데이터 | 프로젝트 이름, 프로젝트 유형, 캡션 또는 날짜별 문구, D-day 설정, 선택한 미디어 참조 정보, 편집 구간, 클립 순서, 볼륨 설정, 회전 및 화면 비율, 내보내기 설정, 백업 메타데이터, 앱 설정값, 최초 가이드 표시 여부 | 앱 내부 저장소에 저장 | SEMO 자체 서버로 자동 전송하지 않음 |
| 내보내기·백업·진단 파일 | 내보내기 영상 파일, 백업 ZIP, 복원 시 사용자가 선택한 백업 ZIP, 임시 내보내기 파일, 진단 로그 | 사용자 요청에 따라 생성·저장·처리 | 자동 전송하지 않음. 사용자가 직접 파일 앱, 메신저, 클라우드 앱, 이메일 등으로 공유할 수 있음 |
| Firebase Analytics 정보 | 앱 사용 이벤트(예: 화면 이동, 프로젝트 관리, 내보내기, 백업·복원, 기능 사용 등) | Google Firebase Analytics SDK를 통해 수집 | Google Firebase로 전송될 수 있음 |
| Firebase Crashlytics 정보 | 오류 정보, 비정상 종료 정보, 앱 버전, 기기 정보 등 | Google Firebase Crashlytics SDK를 통해 수집 | Google Firebase로 전송될 수 있음 |
| 문의 및 오류 신고 정보 | 사용자가 이메일 또는 외부 오류 신고 양식에 직접 입력한 내용, 첨부한 파일 또는 진단 로그 | 사용자가 직접 제출한 경우에만 처리 | 사용자가 선택한 이메일 서비스 또는 외부 양식 서비스로 전달될 수 있음 |
| Google Play 인앱결제 정보 | 구매 상품 ID, 구매 상태, 구매 승인·복원·환불·취소 관련 상태 | Google Play 결제 시스템을 통해 구매 처리 및 Premium 이용 권한 확인 | SEMO는 이를 자체 서버, Analytics, Crashlytics에 저장·전송하지 않으며, Google Play Billing SDK 내부에서 결제 확인·복원·승인을 위해 처리될 수 있음 |

## 3. 앱 권한 사용 목적

SEMO는 Android 기기에서 다음 권한을 사용할 수 있습니다.

| 권한 | 사용 목적 |
|---|---|
| 사진 및 영상 접근 권한 | 사용자가 선택한 사진·영상을 프로젝트에 추가하고 편집하기 위해 사용합니다. |
| 일부 사진 및 영상 접근 권한 | Android 14 이상에서 사용자가 허용한 미디어만 접근하기 위해 사용합니다. |
| 카메라 권한 | 앱 안에서 사진 또는 영상을 촬영하기 위해 사용합니다. |
| 마이크 권한 | 영상 촬영 시 오디오를 함께 기록하기 위해 사용합니다. |
| 인터넷 권한 | Firebase Analytics, Firebase Crashlytics, Google Play 인앱결제, 구매상태 확인 및 복원, 외부 오류 신고 링크 등 네트워크 기능을 위해 사용합니다. |
| 네트워크 상태 권한 | 네트워크 상태 확인 및 Firebase SDK 동작을 위해 사용합니다. |
| 광고 ID 관련 권한 | Firebase Analytics 및 Google Play Services 측정 기능에서 사용할 수 있습니다. 현재 SEMO는 앱 내 광고 표시 목적으로 광고 ID를 사용하지 않습니다. |
| 저장소 관련 권한 | 일부 Android 버전에서 사진·영상 접근 또는 촬영 파일 처리를 위해 사용합니다. |

SEMO는 전체 파일 접근 권한인 `MANAGE_EXTERNAL_STORAGE`를 사용하지 않습니다. SEMO는 알림 권한인 `POST_NOTIFICATIONS`를 사용하지 않습니다.

## 4. 14세 미만 아동의 개인정보 처리에 관한 사항

SEMO는 14세 미만 아동을 직접 대상으로 하는 서비스가 아닙니다. SEMO는 보호자 또는 일반 사용자가 개인적인 사진·영상 기록을 만들기 위한 앱입니다.

사용자는 아동의 사진이나 영상을 앱에 추가할 수 있으나, 해당 사진과 영상은 기본적으로 사용자의 기기 내부에서 처리되며 SEMO 자체 서버로 자동 업로드되지 않습니다. SEMO는 원칙적으로 14세 미만 아동의 개인정보를 수집하지 않습니다. 만약 만 14세 미만 아동의 개인정보(이메일 문의 등)를 수집하게 되는 경우, 법정대리인의 동의를 구하는 등 법령에 따른 절차를 준수하겠습니다.

## 5. 개인정보의 처리 및 보유 기간

### 5.1 기기 내부 데이터

프로젝트 정보, 편집 정보, 캡션, D-day 설정, 보존 클립, 썸네일 등은 사용자의 기기 내부에 저장됩니다. 사용자는 앱 내 프로젝트 삭제, 클립 삭제, 저장공간 관리 기능을 통해 관련 데이터를 삭제할 수 있습니다. 앱을 삭제하면 앱 내부 저장소에 저장된 데이터는 일반적으로 함께 삭제됩니다.

### 5.2 앱 외부에 저장된 파일

다음 파일은 앱 외부 저장소에 저장될 수 있으며, 앱을 삭제해도 남을 수 있습니다.

- Downloads/SEMO 폴더에 저장된 내보내기 영상
- Downloads/SEMO 폴더에 저장된 백업 ZIP
- Downloads/SEMO 폴더에 저장된 진단 로그
- 갤러리 또는 DCIM/SEMO 폴더에 저장된 촬영 파일

이러한 파일은 사용자가 파일 앱, 갤러리 앱 또는 저장공간 관리 기능을 통해 직접 삭제해야 합니다.

### 5.3 Firebase 데이터

Firebase Analytics 및 Firebase Crashlytics를 통해 수집된 데이터는 Google Firebase의 보관 정책과 설정에 따라 보관 및 삭제될 수 있습니다.

SEMO는 계정 기반 앱이 아니므로 특정 사용자의 Firebase 데이터를 식별하거나 삭제하는 데 제한이 있을 수 있습니다.

### 5.4 Google Play 인앱결제 정보

Google Play를 통한 구매 내역과 거래 정보는 Google의 정책 및 관련 법령에 따라 Google Play에서 보관될 수 있습니다. SEMO는 현재 구매 토큰이나 구매 내역을 자체 서버에 별도로 저장하지 않습니다.

SEMO는 Premium 상태를 빠르게 확인하기 위해 구매 확인 결과 또는 Premium 활성 상태를 사용자의 기기 내부에 저장할 수 있습니다. 해당 로컬 정보는 앱 데이터 삭제 또는 앱 삭제 시 삭제될 수 있으나, 동일한 Google 계정의 유효한 구매 내역은 Google Play를 통해 다시 확인하거나 복원할 수 있습니다.

## 6. 개인정보의 파기 절차 및 방법

SEMO는 자체 서버에 사용자의 사진·영상 원본, 프로젝트 정보, 캡션, D-day 설정 등을 저장하지 않습니다. 따라서 SEMO 자체 서버에서 별도로 파기할 사용자 콘텐츠 데이터는 없습니다.

다만 앱 기능 제공 과정에서 기기 내부 또는 앱 외부 저장소에 생성·저장되는 데이터는 다음 방식으로 삭제할 수 있습니다.

| 데이터 구분 | 파기 또는 삭제 방법 |
|---|---|
| 앱 내부 프로젝트 및 편집 데이터 | 앱 내 프로젝트 삭제, 클립 삭제, 저장공간 관리 기능 또는 앱 삭제를 통해 삭제할 수 있습니다. |
| 보존 클립, 썸네일, 변환 파일, 임시 파일 | 앱 내 저장공간 관리 기능 또는 앱 삭제를 통해 삭제할 수 있습니다. |
| Downloads/SEMO에 저장된 내보내기 영상, 백업 ZIP, 진단 로그 파일 | 파일 앱 또는 저장공간 관리 기능을 통해 사용자가 직접 삭제할 수 있습니다. |
| 갤러리 또는 DCIM/SEMO에 저장된 촬영 파일 | 갤러리 앱 또는 파일 앱을 통해 사용자가 직접 삭제할 수 있습니다. |
| Firebase Analytics 및 Crashlytics 데이터 | Google Firebase의 보관 및 삭제 정책에 따라 처리됩니다. SEMO는 계정 기반 앱이 아니므로 특정 사용자의 Firebase 데이터를 식별하거나 삭제하는 데 제한이 있을 수 있습니다. |
| 기기 내부 Premium 상태 캐시 | 앱 데이터 삭제 또는 앱 삭제를 통해 삭제할 수 있습니다. |
| Google Play 구매 내역 | Google Play 및 관련 법령의 보관·삭제 정책에 따라 처리됩니다. 결제 취소와 환불은 Google Play의 주문 및 결제 관리 절차를 따릅니다. |

## 7. 개인정보의 제3자 제공에 관한 사항

SEMO는 사용자의 사진·영상 원본, 프로젝트 데이터, 캡션, D-day 설정 등을 SEMO 자체 서버로 자동 전송하지 않으며, 이를 제3자에게 판매하거나 제공하지 않습니다.

다만 사용자가 앱의 공유 기능 또는 기기의 파일 공유 기능을 이용하는 경우, 내보내기 영상, 백업 ZIP, 진단 로그, 오류 신고에 첨부하거나 입력한 정보가 사용자가 선택한 외부 앱 또는 서비스로 전달될 수 있습니다. 이 경우 데이터 처리는 사용자가 선택한 외부 앱 또는 서비스의 개인정보처리방침을 따릅니다.

Google Firebase 및 Google Play 관련 처리는 아래 “8. 개인정보 처리업무의 위탁” 및 “9. 개인정보의 국외 이전” 항목에서 안내합니다.

## 8. 개인정보 처리업무의 위탁

SEMO는 앱 사용 분석, 오류 분석 및 앱 안정성 개선, Google Play 인앱결제 처리를 위해 다음과 같이 개인정보 처리업무를 외부 서비스에 위탁할 수 있습니다.

| 수탁자 | 위탁 업무 | 처리될 수 있는 정보 |
|---|---|---|
